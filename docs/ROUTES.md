# 路由設計文件 (ROUTES)

本文件定義了「個人記帳簿」系統的所有 URL 路由、HTTP 方法以及對應的處理邏輯與 Jinja2 模板。

## 1. 路由總覽表格

| 功能 | HTTP 方法 | URL 路徑 | 對應模板 | 說明 |
| --- | --- | --- | --- | --- |
| 檢視首頁 | GET | `/` | `templates/index.html` | 顯示所有收支清單與當月統計 |
| 新增收支頁面 | GET | `/record/add` | `templates/form.html` | 顯示新增收支的空白表單 |
| 處理新增收支 | POST | `/record/add` | — | 接收表單，驗證後存入 DB，重導向至首頁 |
| 編輯收支頁面 | GET | `/record/edit/<int:id>` | `templates/form.html` | 顯示表單並帶入原有資料 |
| 處理編輯收支 | POST | `/record/edit/<int:id>` | — | 接收表單，更新 DB，重導向至首頁 |
| 處理刪除收支 | POST | `/record/delete/<int:id>`| — | 刪除指定收支，重導向至首頁 |

## 2. 每個路由的詳細說明

### 檢視首頁 (`GET /`)
- **輸入**：無
- **處理邏輯**：
  - 查詢資料庫中所有的收支紀錄 (依日期降冪排序)
  - 計算當月總支出與總收入
- **輸出**：渲染 `index.html` 並傳入 `records`, `total_income`, `total_expense`
- **錯誤處理**：無特殊錯誤，若無資料則顯示「尚無紀錄」

### 新增收支頁面 (`GET /record/add`)
- **輸入**：無
- **處理邏輯**：無特殊邏輯
- **輸出**：渲染 `form.html`，以「新增模式」呈現

### 處理新增收支 (`POST /record/add`)
- **輸入**：表單欄位（`type` 收支類型, `amount` 金額, `date` 日期, `category` 分類, `note` 備註）
- **處理邏輯**：
  - 驗證必填欄位是否皆有值且格式正確
  - 實例化 `Record` 模型並寫入資料庫
- **輸出**：重導向至 `/`
- **錯誤處理**：驗證失敗時，重新渲染 `form.html` 並顯示錯誤訊息

### 編輯收支頁面 (`GET /record/edit/<int:id>`)
- **輸入**：URL 參數 `id`
- **處理邏輯**：
  - 以 `id` 從資料庫查詢該筆收支紀錄
- **輸出**：渲染 `form.html`，以「編輯模式」呈現，並將原有資料填入表單欄位
- **錯誤處理**：若找不到該筆資料，回傳 404 Not Found 或重導向至 `/` 並提示錯誤

### 處理編輯收支 (`POST /record/edit/<int:id>`)
- **輸入**：表單欄位（同新增）
- **處理邏輯**：
  - 以 `id` 查詢紀錄，若不存在則回傳錯誤
  - 驗證表單資料
  - 更新該筆紀錄的屬性，寫入資料庫
- **輸出**：重導向至 `/`
- **錯誤處理**：驗證失敗時，重新渲染 `form.html` 顯示錯誤

### 處理刪除收支 (`POST /record/delete/<int:id>`)
- **輸入**：URL 參數 `id`
- **處理邏輯**：
  - 查詢並刪除該筆資料
- **輸出**：重導向至 `/`
- **錯誤處理**：找不到紀錄則忽略或重導向至 `/`

## 3. Jinja2 模板清單

- `templates/base.html`：基礎骨架，包含 HTML `<head>` 區塊、全站導覽列與引入共用 CSS。
- `templates/index.html`：繼承自 `base.html`，負責顯示首頁的統計卡片與收支列表表格。
- `templates/form.html`：繼承自 `base.html`，作為共用表單，根據傳入參數決定顯示「新增」或「編輯」標題，並動態設定表單的 action。
