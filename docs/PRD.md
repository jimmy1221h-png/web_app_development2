# PRD：個人記帳簿

## 1. 專案概述

**專案名稱**：個人記帳簿 (Personal Expense Tracker)
**專案描述**：一個簡單易用的個人財務管理系統，幫助使用者快速記錄每日的收入與支出，並透過直觀的報表了解自身的財務狀況。
**目標用戶**：想開始培養記帳習慣的學生、上班族，或需要輕量化追蹤日常開銷的任何人。
**核心價值主張**：介面簡潔、操作直覺，降低記帳的阻力，讓追蹤金流變得輕鬆無負擔。

## 2. 功能需求

以下為系統的主要功能，以使用者故事 (User Story) 描述：

1. **新增收支記錄**
   - *作為一般用戶，我希望可以新增一筆支出或收入（包含金額、日期、分類、備註），以便記錄我的財務狀況。*
2. **檢視收支明細**
   - *作為一般用戶，我希望可以瀏覽按日期排序的收支紀錄清單，以便確認每一筆花費並檢查是否有漏記。*
3. **分類管理與標籤**
   - *作為一般用戶，我希望可以在記帳時選擇收支的分類（如餐飲、交通、薪水），以便了解各類別的金錢流向。*
4. **簡易統計圖表 / 報表**
   - *作為一般用戶，我希望可以查看當月收支的總計與各分類佔比，以便直觀地掌握本月的財務狀況。*
5. **編輯與刪除記錄**
   - *作為一般用戶，我希望可以修改或刪除已建立的收支記錄，以便更正輸入錯誤的資料。*

## 3. 非功能需求

- **技術限制**：後端使用 Python Flask，前端使用 Jinja2 模板引擎與原生 HTML/CSS/JS 實作，資料庫使用 SQLite。
- **效能考量**：頁面操作需流暢，讀取與寫入資料的延遲應保持在使用者幾乎感覺不到的範圍（<1秒）。
- **安全考量**：防範基礎的 SQL Injection (使用 Parameterized query 或 ORM)。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**
  - 新增收入/支出記錄功能。
  - 檢視所有收支紀錄的列表（首頁）。
  - 內建基礎分類（例如：食、衣、住、行、育、樂、薪資）。
  - 使用 SQLite 儲存與讀取資料。
- **Should Have (應該有)**
  - 修改與刪除現有紀錄的功能。
  - 基本的月度總收支計算。
- **Nice to Have (可以有)**
  - 圓餅圖或長條圖的視覺化統計（如使用 Chart.js）。
  - 自訂新增/刪除分類。
  - 使用者註冊與登入系統（多使用者支援）。
